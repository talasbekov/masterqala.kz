# Первая бесплатная версия: техническая спецификация

Статус: целевое поведение бесплатного пилота. В текущем коде отдельный бесплатный режим ещё не реализован.

## 1. Цель

Запустить MasterQala.kz в одном городе с реальными клиентами, мастерами, заказами, матчингом и поддержкой, но без списания денег платформой:

- клиент не оплачивает сервисный сбор и выезд через платформу;
- мастер не покупает и не расходует lead-кредиты;
- платформа не выполняет выплаты мастерам;
- стоимость работ клиент и мастер согласуют в приложении, а фактический расчёт между ними происходит вне платформы;
- система продолжает собирать номинальные цены и воронку для расчёта будущей экономики.

## 2. Почему mock payments недостаточно

Текущий `MockPaymentProvider` имитирует успешную работу провайдера и создаёт записи `HOLD`, `CAPTURE`, `VOID`, charge, payout и refund. Плановый отклик также реально уменьшает баланс lead-кредитов.

Следовательно:

- `mock` означает «техническая имитация платной модели»;
- `free pilot` означает «финансовые обязательства платформы отключены бизнес-правилами».

Это должны быть разные режимы.

## 3. Режим коммерции

Рекомендуется одна конфигурация вместо набора противоречащих boolean-флагов:

```env
COMMERCIAL_MODE=FREE_PILOT
```

Допустимые значения:

| Режим | Назначение |
|---|---|
| `FREE_PILOT` | первая публичная бесплатная версия |
| `PAID_MOCK` | разработка и тестирование платной state machine |
| `PAID_LIVE` | production с реальным платежным провайдером |

Валидация значения выполняется при старте API. Неизвестное значение должно останавливать запуск.

## 4. Поведение `FREE_PILOT`

### 4.1 Срочная заявка

Сохраняются:

- расчёт `calloutPrice` и `serviceFee` как номинальных значений;
- поиск и волновой матчинг;
- офферы;
- статусы;
- предложение и подтверждение цены работ;
- споры, отзывы, отмены и санкции;
- realtime и геопозиция.

Отключаются:

- вызов `payments.hold()` при создании заявки;
- `payments.capture()` при принятии;
- `payments.void()` при отмене или отсутствии мастеров;
- денежное начисление `Accrual` мастеру;
- доступный к выводу баланс и вывод средств.

UI показывает:

- «Выезд: бесплатно на период пилота»;
- «Сервисный сбор: 0 ₸»;
- пояснение, что стоимость работ согласуется после осмотра;
- отсутствие выбора платёжного метода платформы.

Номинальная рассчитанная стоимость может храниться в заказе для аналитики, но не должна показываться как сумма к оплате.

### 4.2 Плановая заявка

Сохраняются:

- публикация;
- лента мастеров;
- максимум пять предложений;
- ставка с ценой, сроком и комментарием;
- выбор мастера;
- подтверждение;
- выполнение, отзыв и спор.

Отключаются:

- проверка достаточности lead-кредитов;
- уменьшение `LeadCreditAccount.balance`;
- создание операции `LeadCreditTransaction` типа `SPEND`;
- покупка пакетов lead-кредитов в пользовательском интерфейсе.

UI показывает «Отклик бесплатный в период пилота».

### 4.3 Кошелёк мастера

В `FREE_PILOT`:

- раздел денежных начислений скрывается либо отображается как недоступный;
- вывод средств запрещён на backend независимо от состояния frontend;
- клиент рассчитывается с мастером вне платформы;
- платформа не гарантирует и не хранит факт внешнего расчёта в качестве платёжной транзакции.

## 5. Архитектурная реализация

### 5.1 Сервис режима

Добавить централизованный сервис, например:

```text
apps/api/src/commercial-mode/
  commercial-mode.module.ts
  commercial-mode.service.ts
  commercial-mode.types.ts
```

Минимальный интерфейс:

```ts
type CommercialMode = 'FREE_PILOT' | 'PAID_MOCK' | 'PAID_LIVE';

class CommercialModeService {
  mode(): CommercialMode;
  isFreePilot(): boolean;
  paymentsEnabled(): boolean;
  leadCreditsEnabled(): boolean;
  payoutsEnabled(): boolean;
}
```

Бизнес-сервисы не должны напрямую читать `process.env`.

### 5.2 Точки изменения backend

Срочный режим:

- `OrdersService.create` — пропустить HOLD;
- `OrdersService.accept` — пропустить CAPTURE;
- `markNoMasters`, отмены и повторный поиск — не вызывать VOID/HOLD;
- `closeOrder` и платные отмены — не создавать денежное начисление;
- endpoints кошелька/вывода — вернуть `403` или доменную ошибку «Недоступно в бесплатном пилоте».

Плановый режим:

- `PlannedOrdersService.placeBid` — в бесплатном режиме создать ставку без списания кредита;
- endpoints покупки кредитов — запретить;
- seed не обязан начислять стартовые кредиты для работы пилота.

API-ответы:

Добавить публичный объект конфигурации:

```json
{
  "commercialMode": "FREE_PILOT",
  "paymentsEnabled": false,
  "leadCreditsEnabled": false,
  "payoutsEnabled": false
}
```

Frontend использует этот ответ для текста и скрытия коммерческих элементов. Backend остаётся источником истины и самостоятельно блокирует запрещённые действия.

### 5.3 Данные

В первой итерации миграция БД не обязательна. Используются существующие поля:

- `calloutPrice` — номинальная расчётная стоимость выезда;
- `serviceFee` — номинальный будущий сбор;
- `workPrice` — согласованная цена работ;
- отсутствие `PaymentTransaction` означает отсутствие платформенной оплаты.

Для аналитической прозрачности рекомендуется позднее добавить в заказ снимок режима:

```prisma
commercialMode String
```

Это исключит неоднозначность после переключения production из бесплатного режима в платный.

## 6. Переходы статусов

Бесплатный режим не должен менять state machine заявок. Меняется только побочный финансовый эффект перехода.

Пример:

```text
SEARCHING → ACCEPTED
FREE_PILOT: назначить мастера, закрыть остальные офферы
PAID_*: то же самое + CAPTURE
```

Такой подход позволяет проверить операционную модель на реальных пользователях и позже включить оплату без переписывания заказа.

## 7. Тесты

Обязательная матрица:

| Сценарий | FREE_PILOT | PAID_MOCK |
|---|---|---|
| Создание срочной заявки | нет PaymentTransaction | создаётся HOLD |
| Принятие мастером | нет CAPTURE | создаётся CAPTURE |
| Нет мастеров | нет VOID | создаётся VOID |
| Закрытие | нет Accrual | создаётся Accrual |
| Плановая ставка | баланс не меняется | списывается 1 кредит |
| Покупка кредитов | запрещена | mock charge |
| Вывод | запрещён | mock payout |

Нужны unit-тесты `CommercialModeService` и e2e для обоих режимов.

## 8. Критерии готовности бесплатной версии

Версия считается бесплатной только когда одновременно выполнено следующее:

1. `COMMERCIAL_MODE=FREE_PILOT` валидируется при старте.
2. Клиент может пройти полный срочный сценарий без способа оплаты.
3. Мастер может откликнуться на плановую заявку с нулевым балансом кредитов.
4. Ни один endpoint не создаёт `PaymentTransaction`, `Accrual` или платную lead-credit операцию.
5. Вывод и покупка кредитов заблокированы backend.
6. UI во всех местах явно сообщает о бесплатном пилоте.
7. E2E подтверждает отсутствие финансовых записей.
8. В документации описан порядок переключения в `PAID_MOCK` и `PAID_LIVE`.

## 9. Что не отключается

Бесплатный пилот не означает упрощение контроля качества. Обязательными остаются:

- ручная верификация мастеров;
- журнал решений оператора;
- ограничения категорий и геозоны;
- санкции за отмены;
- споры;
- отзывы;
- аудит статусов;
- защита персональных данных;
- мониторинг и резервное копирование.
