# State machines заявок

Документ описывает фактические переходы, реализованные в backend. Бесплатный пилот сохраняет те же статусы; меняются только финансовые побочные эффекты.

## 1. Срочная заявка

### 1.1 Основной сценарий

```text
CREATED
  │ создание HOLD или пропуск в FREE_PILOT
  ▼
SEARCHING
  │ первый мастер успешно принял оффер
  ▼
ACCEPTED
  │ мастер нажал «Еду»
  ▼
MASTER_ON_WAY
  │ мастер нажал «На месте»
  ▼
INSPECTION
  │ мастер предложил цену
  ▼
AWAITING_PRICE_CONFIRM
  │ клиент подтвердил цену
  ▼
IN_PROGRESS
  │ мастер отметил выполнение
  ▼
DONE
  │ клиент подтвердил или прошло 24 часа без открытого спора
  ▼
CLOSED
```

### 1.2 Альтернативные ветки

```text
SEARCHING ── исчерпаны 3 волны ──► NO_MASTERS
NO_MASTERS ── повторить поиск ────► SEARCHING

CREATED / SEARCHING / NO_MASTERS
  └─ отмена клиентом ─────────────► CANCELLED_BY_CLIENT

ACCEPTED / MASTER_ON_WAY
  ├─ отмена клиентом ─────────────► CANCELLED_BY_CLIENT
  └─ отмена мастером ─────────────► SEARCHING

AWAITING_PRICE_CONFIRM
  ├─ клиент отклонил цену ────────► CANCELLED_BY_CLIENT
  ├─ клиент отменил ──────────────► CANCELLED_BY_CLIENT
  └─ таймаут 15 минут ────────────► CANCELLED_BY_CLIENT

DONE ── открыт спор ──────────────► DISPUTE или заморозка автозакрытия согласно dispute-сервису
```

### 1.3 Ограничения переходов

- У клиента может быть только одна активная срочная заявка.
- У мастера может быть только одна активная срочная заявка.
- `SEARCHING → ACCEPTED` выполняется атомарно; второй мастер получает `409`.
- На цену даётся 15 минут.
- Автозакрытие после `DONE` — 24 часа.
- Открытый спор останавливает автозакрытие.
- Отмена мастером в `ACCEPTED` или `MASTER_ON_WAY` возвращает заказ в поиск и увеличивает номер попытки.

### 1.4 Финансовые эффекты

| Переход | PAID_MOCK / PAID_LIVE | FREE_PILOT |
|---|---|---|
| создание | HOLD стоимости выезда | нет транзакции |
| `SEARCHING → ACCEPTED` | CAPTURE | нет транзакции |
| `SEARCHING → NO_MASTERS` | VOID | нет транзакции |
| ранняя отмена клиента | VOID | нет транзакции |
| платная отмена/таймаут цены | компенсация мастеру | нет начисления |
| `DONE → CLOSED` | компенсация мастеру | нет начисления |

### 1.5 Замечание по enum

В Prisma присутствует `CANCELLED_BY_MASTER`, но текущая реализация ранней отмены мастером не переводит срочную заявку в этот статус: она возвращает её в `SEARCHING`. Статус следует либо удалить как неиспользуемый, либо определить отдельный терминальный сценарий и покрыть тестами.

## 2. Плановая заявка

### 2.1 Основной сценарий

```text
CREATED
  │ сервис создаёт и сразу публикует
  ▼
PUBLISHED
  │ клиент выбрал ставку
  ▼
MASTER_SELECTED
  │ мастер подтвердил в течение 2 часов
  ▼
CONFIRMED
  │ мастер отметил прибытие/начало
  ▼
IN_PROGRESS
  │ мастер отметил выполнение
  ▼
DONE
  │ клиент подтвердил или прошло 24 часа без открытого спора
  ▼
CLOSED
```

В текущем сервисе запись фактически создаётся сразу со статусом `PUBLISHED`, поэтому `CREATED` существует в enum и логике отмен, но обычно не наблюдается как отдельный сохранённый этап.

### 2.2 Выбор и возврат к публикации

```text
MASTER_SELECTED
  ├─ мастер отклонил ─────────────► PUBLISHED
  └─ мастер не ответил 2 часа ─────► PUBLISHED

CONFIRMED / IN_PROGRESS
  └─ мастер отменил ───────────────► PUBLISHED
```

При возврате очищаются назначенный мастер, выбранная ставка и временные метки выбора/подтверждения. Отмена подтверждённым мастером вызывает санкцию.

### 2.3 Отмена клиента

```text
CREATED / PUBLISHED
  └─ отмена клиентом ──────────────► CANCELLED_BY_CLIENT

MASTER_SELECTED / CONFIRMED / IN_PROGRESS
  └─ отмена клиентом ──────────────► CANCELLED_BY_CLIENT
```

В платном режиме после выбора мастера клиентская отмена возвращает мастеру один lead-кредит. В бесплатном пилоте кредитная операция отсутствует.

### 2.4 Истечение и спор

- Если наступил `slotStart`, заявка всё ещё `PUBLISHED` и не получила ни одной ставки, она переходит в `EXPIRED`.
- После `DONE` автозакрытие происходит через 24 часа.
- Открытый спор останавливает автозакрытие.
- Enum содержит `CANCELLED_BY_MASTER`, но текущая отмена мастером возвращает плановую заявку в `PUBLISHED`, а не завершает её этим статусом.

### 2.5 Lead-кредиты

| Действие | PAID_MOCK / PAID_LIVE | FREE_PILOT |
|---|---|---|
| мастер делает ставку | списывается 1 кредит | бесплатно |
| клиент отменяет после выбора | возврат 1 кредита выбранному мастеру | нет операции |
| покупка пакета | charge провайдера | запрещена |

## 3. Общие правила реализации

- Каждый переход должен проверять владельца действия: клиент, назначенный мастер или оператор.
- Переходы выполняются атомарным условным обновлением по исходному статусу.
- Повторный или запоздалый запрос должен завершаться `409 Conflict`, а не создавать дублирующий эффект.
- Очереди и таймауты должны быть идемпотентными: если статус уже изменился, обработчик завершается без побочных действий.
- Frontend не является источником истины для разрешённых переходов.
- При изменении enum необходимо одновременно обновить сервисы, UI-маппинг, тесты, аналитику и этот документ.
