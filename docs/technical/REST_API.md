# REST API MasterQala.kz

Этот документ фиксирует HTTP-контракт backend MasterQala.kz после реализации бесплатного пилота в ветке `feat/free-pilot-mode` (PR #4).

До объединения PR #4 ветка `main` может не содержать часть описанных capability-флагов и поведения `FREE_PILOT`.

## 1. Базовые правила

- базовый URL: `/api/v1`;
- стандартный формат запроса и ответа: JSON;
- загрузка файлов: `multipart/form-data`;
- выдача защищённых изображений и документов: бинарный поток;
- защищённые маршруты используют `Authorization: Bearer <accessToken>`;
- JWT действует 30 дней;
- суммы передаются целыми числами в тенге;
- даты и время передаются в ISO 8601;
- координаты: `lat` от `-90` до `90`, `lng` от `-180` до `180`;
- глобальная валидация NestJS использует `whitelist: true` и `transform: true`;
- неизвестные поля удаляются из body до передачи в контроллер;
- CORS в текущей реализации разрешает любой origin.

Пример заголовков:

```http
Authorization: Bearer eyJ...
Content-Type: application/json
```

## 2. Пользовательская модель доступа

В `User.role` сейчас существуют:

- `CLIENT` — обычный пользователь; после активации анкеты он также может выполнять действия мастера;
- `OPERATOR` — оператор административного контура.

Отдельной роли `MASTER` в `User.role` нет. Доступ мастера определяется наличием и состоянием `MasterProfile`, оффером, назначением на заявку и проверками в сервисах.

## 3. Стандартные ошибки

NestJS возвращает стандартный error body, например:

```json
{
  "statusCode": 403,
  "message": "Вывод средств недоступен в бесплатном пилоте",
  "error": "Forbidden"
}
```

| HTTP | Когда используется |
|---|---|
| `400` | невалидный DTO, телефон, файл, категория или параметры |
| `401` | отсутствует, истёк или невалиден JWT |
| `403` | нет доступа к ресурсу или функция отключена в `FREE_PILOT` |
| `404` | ресурс не найден |
| `409` | конфликт состояния, повторное действие или проигранная гонка |
| `422` | бизнес-условие не выполнено: нет мастеров, недостаточно средств/кредитов, лимит или блокировка |
| `429` | превышен лимит запроса SMS-кода |
| `503` | неопределённый результат операции внешнего провайдера |

Машинных кодов ошибок пока нет. Клиент не должен надолго привязывать бизнес-логику к русскому тексту `message`.

## 4. Коммерческий режим

Поддерживаемые значения окружения:

- `FREE_PILOT`;
- `PAID_MOCK`;
- `PAID_LIVE` — зарезервирован, но API намеренно не запускается до подключения реального провайдера.

Если переменная отсутствует, backend использует `PAID_MOCK` для обратной совместимости. Пример окружения пилота использует `FREE_PILOT`.

### `GET /config/public`

Публичный маршрут без JWT.

Ответ бесплатного пилота:

```json
{
  "commercialMode": "FREE_PILOT",
  "paymentsEnabled": false,
  "leadCreditsEnabled": false,
  "payoutsEnabled": false
}
```

Ответ mock-платного режима:

```json
{
  "commercialMode": "PAID_MOCK",
  "paymentsEnabled": true,
  "leadCreditsEnabled": true,
  "payoutsEnabled": true
}
```

Capability-флаги определяют доступность создания новых коммерческих операций. Поведение уже созданной заявки определяется её сохранённым `commercialMode`, а не текущим env.

## 6. Авторизация

### `POST /auth/request-code`

Запрашивает SMS-код.

Body:

```json
{
  "phone": "+77001234567"
}
```

Правила:

- номер нормализуется backend;
- код состоит из 6 цифр;
- срок действия — 5 минут;
- максимум 3 отправки на номер за 10 минут.

Успех: `204 No Content`.

Возможные ошибки:

- `400` — неверный формат телефона;
- `429` — превышен лимит отправок.

### `POST /auth/verify-code`

Body:

```json
{
  "phone": "+77001234567",
  "code": "123456"
}
```

Правила:

- `code` должен иметь длину ровно 6 символов;
- максимум 5 неверных попыток для актуальной записи;
- при первом успешном входе пользователь создаётся автоматически.

Ответ:

```json
{
  "accessToken": "eyJ...",
  "user": {
    "id": "uuid",
    "phone": "+77001234567",
    "name": null,
    "role": "CLIENT"
  }
}
```

## 10. Анкета мастера

### `POST /masters/application`

Требует JWT.

```json
{
  "fullName": "Иван Иванов",
  "iin": "900101123456",
  "district": "Есиль",
  "experienceYears": 5,
  "categoryIds": ["uuid"]
}
```

Ограничения:

- `fullName`: 5–150 символов;
- `iin`: ровно 12 цифр;
- `district`: 2–100 символов;
- `experienceYears`: целое число `0..60`;
- `categoryIds`: минимум один элемент;
- все категории должны существовать.

Повторная отправка разрешена для ранее отклонённой анкеты или `NEEDS_INFO`. Статусы `PENDING_REVIEW` и `ACTIVE` дают `409`.

### `GET /masters/application`

Возвращает собственную анкету, категории, документы и `latestDecisionComment`.

### `POST /masters/application/documents`

Требует JWT и `multipart/form-data`.

Поля:

- `file` — JPEG, PNG или PDF;
- `type` — значение Prisma enum `DocumentType`.

Ограничения:

- максимум 10 МБ;
- сначала должна существовать анкета;
- загрузка разрешена только в `PENDING_REVIEW` или `NEEDS_INFO`.

## 11. Универсальная загрузка изображений

### `POST /uploads`

Требует JWT и `multipart/form-data`.

- поле: `file`;
- форматы: JPEG или PNG;
- размер: максимум 10 МБ.

Ответ:

```json
{
  "path": "relative/storage/path.jpg"
}
```

`path` передаётся в `photoPaths` при создании срочной или плановой заявки. Прямого публичного URL нет: файлы выдаются через защищённые маршруты заявок.

### 12.3 Создание

Body `POST /orders` расширяет preview:

| Поле | Тип | Ограничение |
|---|---|---|
| `categoryId` | UUID | обязательно |
| `lat` / `lng` | number | обязательно |
| `description` | string | обязательно, максимум 2000 |
| `address` | string | обязательно, максимум 500 |
| `district` | string | обязательно, максимум 200 |
| `entrance` | string | optional, максимум 50 |
| `floor` | string | optional, максимум 50 |
| `apartment` | string | optional, максимум 50 |
| `addressComment` | string | optional, максимум 500 |
| `photoPaths` | string[] | optional, максимум 5 |

Ключевые ограничения:

- одна активная срочная заявка на клиента;
- категория должна существовать;
- при отсутствии мастеров создание возвращает `422`;
- принятие защищено атомарным переходом; проигравший мастер получает `409`;
- принимать может только мастер с `PENDING`-оффером текущей попытки;
- цена работ подтверждается клиентом в течение 15 минут;
- `DONE` автоматически закрывается через 24 часа при отсутствии открытого спора.

### 12.4 Представление цены в бесплатном пилоте

Для заявки с `commercialMode=FREE_PILOT` HTTP-ответы показывают:

- `calloutPrice: 0`;
- `serviceFee: 0`;
- `freePilot: true`;
- номинальные значения могут возвращаться как `nominalCalloutPrice` и `nominalServiceFee` для аналитики и объяснения расчёта.

`workPrice` — цена работ, которую мастер предлагает после осмотра. Платформа не принимает этот платёж: расчёт происходит напрямую между клиентом и мастером.

Режим определяется сохранённым `Order.commercialMode`. Переключение env не меняет уже созданную заявку.

### 13.2 Создание

```json
{
  "categoryId": "uuid",
  "description": "Установить смеситель",
  "address": "Астана, ...",
  "district": "Есиль",
  "slotStart": "2026-07-25T10:00:00+05:00",
  "slotEnd": "2026-07-25T12:00:00+05:00",
  "budget": 15000,
  "photoPaths": []
}
```

Ограничения:

- `slotStart` и `slotEnd` — ISO 8601;
- начало должно быть в будущем и не дальше 14 дней;
- конец должен быть позже начала;
- `budget` — optional, целое число от 0;
- максимум 5 изображений.

### 13.3 Отклик мастера

`POST /planned-orders/:id/bids`:

```json
{
  "price": 12000,
  "term": "Сегодня до 18:00",
  "comment": "Материалы оплачиваются отдельно"
}
```

Ограничения:

- `price`: целое число от 1;
- `term`: обязательно, максимум 200;
- `comment`: optional, максимум 1000;
- максимум 5 откликов на заявку;
- один отклик одного мастера;
- заблокированный мастер не может откликаться.

Коммерческое поведение определяется `PlannedOrder.commercialMode`:

- `FREE_PILOT` — отклик не требует баланса и не создаёт `SPEND`/`REFUND`;
- `PAID_MOCK` — отклик требует и списывает 1 lead-кредит.

Поэтому в одной ленте могут одновременно находиться бесплатные и исторические платные заявки.

### 13.4 Выбор отклика

`POST /planned-orders/:id/select`:

```json
{
  "bidId": "uuid"
}
```

Выбранный мастер должен подтвердить готовность в течение 2 часов. Заявка без единого отклика становится `EXPIRED` при наступлении `slotStart`. `DONE` автоматически закрывается через 24 часа при отсутствии спора.

До назначения сервис скрывает чувствительные данные. Телефон мастера становится доступен клиенту с `CONFIRMED`.

## 16. Споры

Все пользовательские маршруты требуют JWT и участия в соответствующей заявке.

| Метод | Маршрут | Назначение |
|---|---|---|
| `POST` | `/orders/:id/disputes` | открыть спор по срочной заявке |
| `POST` | `/planned-orders/:id/disputes` | открыть спор по плановой заявке |
| `POST` | `/disputes/:id/evidence` | добавить доказательство |
| `PATCH` | `/disputes/:id` | добавить пояснение второй стороны |
| `GET` | `/disputes/:id/evidence/:docPath` | получить доказательство |

Открытие:

```json
{
  "reason": "Описание проблемы"
}
```

`reason` обязателен, максимум 2000 символов.

Правила:

- спор доступен в `IN_PROGRESS`, `DONE` и `CLOSED`;
- после `CLOSED` окно открытия — 48 часов;
- по заявке может существовать только один спор;
- доказательства: JPEG/PNG, максимум 10 МБ;
- пояснение добавляет только вторая сторона;
- закрытый спор нельзя дополнять.

```json
  "refundServiceFee": false,
  "penalizeMaster": true,
  "resolutionNote": "Решение оператора"
}
```

В `FREE_PILOT` backend принудительно сохраняет `refundServiceFee=false` и не вызывает платёжный refund, потому что платформа не принимала сервисный сбор.

## 20. Защищённые файлы

Следующие маршруты не являются публичными static URL:

- `/orders/:id/photos/:photoId`;
- `/planned-orders/:id/photos/:photoId`;
- `/disputes/:id/evidence/:docPath`;
- `/admin/applications/:id/documents/:docId`.

Перед выдачей сервис проверяет участие пользователя или роль оператора. Клиентские приложения не должны сохранять эти URL как бессрочные публичные ссылки.

## 21. Совместимость HTTP и Socket.IO

HTTP является источником полного состояния заявки. Socket.IO используется для уведомления об изменениях и ускорения интерфейса.

Для `FREE_PILOT` оба канала обязаны показывать нулевые `calloutPrice`/`serviceFee`. Контракты realtime описаны в [`WEBSOCKET_EVENTS.md`](./WEBSOCKET_EVENTS.md).

Клиент должен перечитывать ресурс через HTTP:

- после reconnect;
- после неизвестного события;
- при пропущенной последовательности переходов;
- перед необратимым пользовательским действием.

## 22. Что остаётся до стабильного внешнего API

1. Подключить OpenAPI/Swagger и генерировать типизированный SDK.
2. Ввести машинные error codes и единый envelope.
3. Добавить пагинацию истории, анкет и споров.
4. Добавить idempotency key для создания заявок и критичных POST.
5. Ограничить CORS production-origin.
6. Расширить health-check проверками БД, очереди и хранилища.
7. Добавить rate limits не только для SMS, но и для auth, uploads и mutation endpoints.
8. Проверять сигнатуру загруженного файла, а не только multipart MIME.
9. Добавить audit log операторских решений и коммерческих действий.
10. Формально версионировать изменения payload и политику backward compatibility.
