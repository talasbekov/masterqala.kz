# Handoff: MasterQala.kz — редизайн клиента, мастера и панели оператора

## Обзор

Полный редизайн интерфейса платформы MasterQala.kz (казахстанский маркетплейс бытовых услуг «на дом»): клиентское mobile-PWA (срочный и плановый режимы заказа), рабочее приложение мастера (офферы, статусы, ставки, деньги) и desktop-панель оператора (верификация, заказы, споры, выводы). Спроектировано по этапам: UX-аудит → IA/flows → wireframes → hi-fi кликабельные прототипы.

Целевой репозиторий: `github.com/talasbekov/masterqala.kz` (React 19 + TypeScript + Vite + React Router + Tailwind CSS 4 + socket.io-client; API NestJS + Prisma — **не менять**).

## О файлах дизайна

Файлы в этом пакете — **дизайн-референсы, созданные в HTML** (интерактивные прототипы с симуляцией realtime). Это НЕ production-код: задача разработчика — **воссоздать эти экраны в существующем окружении репозитория** (`apps/web`, компоненты `packages/ui`), используя принятые паттерны (Tailwind-токены через `@theme`, TanStack Query, `gate()`-переходы API, WS-события `RealtimeGateway`). Демо-панель слева в прототипах («прыжки» по состояниям) — инструмент просмотра, в продукт не переносится.

## Fidelity

- `Этап 5 — Клиент (mobile).dc.html`, `Этап 5 — Мастер (mobile).dc.html`, `Этап 6 — Оператор (desktop).dc.html` — **hi-fi**: точные цвета, типографика, отступы, состояния и интеракции. Воссоздавать близко к пикселю на компонентах `packages/ui` (расширив их).
- `Этап 4 - Wireframes.dc.html` — **lo-fi**: только структура; финальный вид — в hi-fi файлах.
- `Этап 1-3 — Аудит…` и `…Sitemap и Flows` — документация: полный список экранов, состояний, флоу и требуемых расширений DTO.

Таб-бар: Главная · Заявки · [＋ FAB 52px] · Уведомления · Профиль.
3. **Wizard срочной заявки, 4 шага** (прогресс-полосы 5px): 1) категория (2 колонки, выбранная — заливка `#EAF2F6` + рамка `#166088`); 2) описание + фото до 5 (72×72, крестик удаления); 3) адрес: карта OSM + поле адреса + чипы сохранённых («Дом»/«Работа») + подъезд/этаж/квартира + комментарий; 4) расчёт: карточка «Выезд 3 500 ₸ / в т.ч. сбор 1 400 ₸ / работы — после осмотра», способ оплаты, «отмена до принятия — бесплатно», CTA «Найти мастера · 3 500 ₸».
4. **Поиск** — карта во весь экран, пульсирующие круги у точки клиента, bottom sheet: текст волны («Ищем в радиусе 3 км…» → 6 → 10), таймер, 3 полосы волн, «Отменить — бесплатно». Волны переключаются по WS/таймауту.
5. **Нет мастеров** — возврат оплаты, «Повторить поиск» / «Запланировать».
6. **Мастер найден/едет/на месте** — карта с пином клиента и движущимся маркером мастера, бейдж статуса; шторка: аватар-инициалы, имя + «✓ проверен», ★ рейтинг · N заказов · категория, кнопка звонка (44px круг `#166088`), плашка ETA на `#CFE0E8`, ссылки Поддержка/Правила отмены/Отменить.
7. **Подтверждение цены** — таймер 15:00 в пилюле `#166088`, разбивка выезд+работы=итого, комментарий мастера, последствия отклонения текстом, CTA «Подтвердить 15 000 ₸» + danger-контур «Отклонить».
8. **В работе** — вертикальный степпер 7 шагов (пройдено — зелёный ✓, текущий — `#166088`).
9. **Выполнено → подтверждение → Закрыт** — итог, оценка ★, «Открыть спор (48 ч)».
10. **Плановый wizard, 3 шага** — описание+фото+район (адрес скрыт до выбора); дата (горизонтальные чипы) + слоты 2×2 + бюджет; предпросмотр «как увидят мастера» + «Опубликовать».
11. **Ожидание предложений** — счётчик N из 5, live-карточки ставок (имя, ✓, ★, заказы, опыт, цена, срок, комментарий, бейдж позиции цены: «ниже средней» зелёный / «средняя» синий / «выше средней» оранжевый), кнопки «Сравнить (N)» и «Отменить»; empty-state со спиннером.
12. **Сравнение** — таблица-сетка 2 мастеров построчно (цена/рейтинг/заказы/опыт/срок/комментарий), выбор → **экран подтверждения выбора** (предупреждение: адрес и телефон откроются мастеру, остальные предложения закроются).
13. **Активная плановая** — «ждём подтверждения (до 2 ч)» со спиннером → после подтверждения зелёный баннер + карточка мастера с контактом.

15. **Деталь закрытой заявки** — описание, мастер, разбивка сумм, timeline, «спор — окно истекло».
16. **Спор** — чипы причины, описание, фото; после отправки — статус-точки (отправлен → ждём мастера → решение до 24 ч) + заморозка автозакрытия.
17. **Поддержка** (звонок 7666 + FAQ), **Уведомления**, **Профиль** (переключатель Рус/Қаз/Eng, «Стать мастером»), **Адреса**, **Способы оплаты**.

### Мастер (390×844)
1. **Онбординг 4 шага**: анкета (ФИО/ИИН/опыт) → категории-чипы + геозона на карте (круг радиуса) → документы (слоты 2×2 с бейджем «✓», состояние «файл не загрузился — повторить») → проверка данных → **4 статуса**: на рассмотрении / нужны данные (комментарий оператора + «Дозагрузить») / одобрен / отклонён (причина + «Подать заново»).
2. **«Работа»** — карточка-тумблер «На линии» (свитч 58×32), бейдж соединения, 3 KPI (заказы/заработок/рейтинг), карточка активного заказа (заливка `#166088`), категории, «Ждём заявки…». Таб-бар: Работа · Плановые · [FAB: 🛠, при активном заказе — красный ⚡] · Кошелёк · Профиль.
3. **Входящий оффер** — полноэкранный, фон `#123B52`, звук (AudioContext 880Hz) + вибрация, красный таймер-пилюля, категория 26px, район+расстояние+время в пути, описание, фото, «Вам за выезд 2 100 ₸», **«точный адрес — после принятия»**, белая CTA «Принять (0:47)» + «Пропустить». Истёк — возврат на «Работа» с пояснением.
4. **Активный заказ** — карта + «Навигатор ↗», карточка адреса (полный, с деталями) + клиент с кнопкой звонка, описание+фото+компенсация; **кнопки статусов по одной на статус**, каждая — через bottom-sheet подтверждения с последствиями («Еду», «На месте», «Выполнена» — зелёная). Отмена — danger-sheet: «−2 кредита, приоритет ↓ 24 ч, 3-я отмена за 30 дней = блокировка 7 дней».
5. **Форма цены** — крупная сумма, «что входит» (видит клиент), чипы длительности, итог для клиента + «15 минут на решение» → ожидание → «цена подтверждена» → работа → выполнено → «+2 100 ₸ в кошелёк».
6. **Плановая лента** — баланс ⬡ в шапке, карточки без адреса (категория, район, описание, дата/слот, N/5 ставок, «отклик: 1 ⬡»); состояние 5/5 — «лимит достигнут».
7. **Форма ставки** — цена/срок/комментарий, плашка «Спишется 1 ⬡ · останется N», CTA «Откликнуться · −1 ⬡»; **0 кредитов** — красная плашка + «Купить кредиты» с возвратом в форму.
8. **Lead-кредиты** — баланс ⬡ («кредиты — не деньги»), пакеты 1/500 ₸, 10/5 000, 25/11 000, 60/24 000, история (SPEND/REFUND/PENALTY/PURCHASE).
9. **Кошелёк ₸** — сегмент ₸/⬡ (визуально разделены), тёмно-синяя карточка баланса + «ожидает начисления», «Вывести · от 5 000 ₸», история со статусами (в обработке/выплачено/отклонён банком + «сумма возвращена»).
10. **Вывод** — сумма, чипы 5 000/20 000/всё, реквизиты = телефон, правила, статус меняется PENDING→PAID.
11. **Профиль** — бейдж «✓ мастер», документы/геозона/споры/поддержка; **состояние блокировки** — красный баннер «доступ ограничен до {дата}, активный заказ можно завершить».
12. **Спор (сторона мастера)** — заявление клиента, одно поле пояснения + фото, возможные исходы.

### Оператор (1440, min-width)
Топ-бар (лого, город/дата, RU ▾, оператор) + левая навигация 216px с бейджами счётчиков (оранжевые для верификации/споров).
1. **Обзор** — ряд «Требует внимания» (4 карточки-кнопки с цветными рамками, ведут в разделы) + ряд метрик + таблица «Поиск без мастера» с кнопкой «Назначить вручную».
2. **Верификация** — master-detail: очередь слева (карточки со статус-пилюлями), деталь справа: анкета в 4 плашках, документы (превью с ⤢), поле комментария, кнопки **Одобрить** (сразу) / **Запросить данные** / **Отклонить** (обе требуют комментария — кнопки полупрозрачны и не срабатывают без него); после решения — цветная плашка результата, статус в очереди обновляется.
3. **Пользователи** — таблица с ролью и заказами, кнопка Заблокировать/Разблокировать (переключается, бейдж «заблокирован»).
4. **Мастера** — таблица: категории, ★, заказы, статус-пилюли (активен·онлайн / офлайн / приоритет ↓ до даты / блокирован до даты).
5. **Заказы** — фильтры, таблица (выбранная строка — заливка `#EAF2F6`) + правый sidebar 400px: статус, timeline с цветными точками, платежи, **«Назначить мастера вручную»** (для зависшего поиска; после — зелёная плашка «назначен, запись в журнале»), внутренний комментарий.
6. **Споры** — список слева; центр — контекст заказа (timeline, платежи, «авто-закрытие заморожено»), заявление клиента + фото, пояснение мастера; правая панель — **два независимых чекбокса** (возврат сбора 1 400 ₸ / санкция мастеру −2 ⬡) + обязательный комментарий + **двухшаговое подтверждение** («Действие необратимо») → статус «решён», заказ закрывается.
7. **Вывод средств** — read-only таблица, телефоны маскированы, статусы; примечание про автоворат при отказе банка.
8. **Журнал** — время / кто (оператор или «система») / что; события: решения по верификации, споры, блокировки, автозакрытия.

## Интеракции и поведение

- Realtime (WS уже есть в API): `offer:new` (звук+вибрация+полноэкранный оффер с дедлайном), `offer:closed`, `order:status` (степпер/карта клиента), `bid:new` (карточка появляется с анимацией slide-up 300ms), `bid:selected`/`bid:closed`, `planned:status`.
- Анимации: slide-up появления карточек/шторок 250–350ms ease; пульс поиска (масштаб 0.6→1.6, fade, 1.8s infinite); спиннеры 0.9s linear; движение маркера мастера — CSS transition 1s linear по координатам. **Поддержать prefers-reduced-motion.**
- Таймеры видимые: оффер (сек), подтверждение цены (15:00 обратный), подтверждение выбора (до 2 ч), «отправить SMS снова через 0:42».
- Критические действия — только через подтверждающий bottom sheet (mobile) или двухшаговое подтверждение (оператор), всегда с текстом последствий. Никаких window.confirm.
- Защита данных: адрес/телефон клиента мастеру — только после принятия (срочный) / подтверждения выбора (плановый); телефон мастера клиенту — после ACCEPTED; в ленте и оффере — только район.
- Состояния на каждый экран: loading/skeleton, empty, error (что/почему/что делать), offline/reconnecting; спец-состояния — см. §10 аудита.
- Карты: реальные тайлы OSM (в прототипе — статичные тайлы z14 Астаны: `tile.openstreetmap.org/14/11442-11443/5477-5478.png`); в продукте — Leaflet/MapLibre с теми же тайлами.
- Доступность: WCAG 2.2 AA, focus-visible (`outline 2px #166088 offset 2px`), touch ≥44px, не только цвет (пилюли всегда с текстом).

## Состояние (клиентская сторона)

Модель фаз в прототипах повторяет серверные enum'ы: срочный `CREATED→SEARCHING(волны 1-3)→ACCEPTED→MASTER_ON_WAY→INSPECTION→AWAITING_PRICE_CONFIRM→IN_PROGRESS→DONE→CLOSED` + `NO_MASTERS`/отмены; плановый `PUBLISHED→MASTER_SELECTED→CONFIRMED→…→CLOSED`/`EXPIRED`. Спор — отдельная сущность (бейдж, не статус заказа). Источник данных — существующие эндпоинты + WS; рефетч active-order при reconnect.
